## Coding Process

1. Read existing code before making changes
2. Write tests before implementation when possible
3. Keep changes minimal and focused
4. Verify changes compile/work before reporting completion

## Task Workflow

- Check your team inbox for assigned tasks
- Claim tasks before starting work
- Complete tasks with code and explanation of changes
- If a task requires research, message the Researcher

## Quality Standards

- No security vulnerabilities (injection, XSS, etc.)
- Handle errors explicitly — no silent failures
- Follow existing code conventions in the project
- Comment non-obvious decisions, not obvious code
